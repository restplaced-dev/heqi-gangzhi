# Codex 交接文件｜河憩缸誌 PWA 原型

## 專案狀態

這是一個目前可放在 GitHub Pages 測試的 PWA 靜態原型。

- Repo 建議名稱：`heqi-gangzhi`
- 目前入口：`index.html`
- 技術：HTML / CSS / Vanilla JavaScript
- 儲存方式：`localStorage`
- 目前沒有登入、後端、資料庫、付款、真正社群 API
- 目標不是一次做正式 App，而是先把「飼主日常紀錄流程」測順

## 產品定位

「河憩缸誌」是給一般水族飼主使用的養魚日常紀錄 App。

核心不是醫療診斷，也不是繁殖管理，而是：

> 把魚真的當寵物照顧，記錄牠在水裡生活的每一天。

主軸包含：

1. 魚缸日常紀錄
2. 魚隻個體檔案
3. 餵食、換水、拍照、清潔、日常觀察
4. 輕社群貼文互動
5. 自訂提醒與魚缸節奏管理

## 非目標與禁忌

請避免把產品改成以下方向：

- 不提供用藥建議
- 不做疾病診斷
- 不把繁殖當主要功能
- 不讓使用者覺得自己在填醫療病歷或養殖報表
- 不使用恐嚇式提醒文案
- 不把水質數據放成唯一主角
- 不鼓勵危險混養、虐待式飼養或未確認的偏方

異常紀錄可以保留，但請用生活化語氣，例如「狀態怪怪的」、「想再觀察」，不要讓 App 看起來像在直接判病。

## 目前已完成

目前原型已有：

- 底部導覽：首頁、魚缸、紀錄、社群、我的
- 首頁：今日魚缸、提醒、最近紀錄
- 魚缸頁：魚缸卡片、魚隻檔案、新增魚缸、新增魚隻
- 紀錄頁：觀察、餵食、換水、拍照、清潔、新魚入缸、狀態怪怪的
- 紀錄備註已改為可留空
- 社群頁：貼文列表、按讚、標籤
- 我的頁：訂閱版方向、重置資料
- PWA manifest 與 service worker

## 目前程式架構

主要檔案：

- `index.html`：App shell、導覽、快速新增 dialog
- `styles.css`：全部樣式
- `app.js`：資料、畫面 render、事件綁定、localStorage

- `state.tanks`
- `state.logs`
- `state.posts`
- `state.reminders`

## 目前最重要的下一步

### 1. 自訂提醒系統

請優先做「每個魚缸可以自己設定提醒」。

需要支援：

- 換水提醒
- 餵食提醒
- 拍照紀錄提醒
- 清潔提醒
- 日常觀察提醒

提醒設定應該屬於單一魚缸，不是全域固定死。

建議 reminder 資料結構：

```js
{
  id: string,
  tankId: string,
  tankName: string,
  type: '換水' | '餵食' | '拍照' | '清潔' | '觀察',
  intervalDays: number,
  time: '21:00',
  lastDoneAt: '2026-06-08',
  nextDueAt: '2026-06-15',
  enabled: true,
  note: '可選填'
}
```

互動邏輯：

- 使用者可以新增提醒
- 使用者可以編輯提醒
- 使用者可以關閉提醒
- 使用者可以按「完成」
- 按完成後，`lastDoneAt` 更新為今天，`nextDueAt` 自動往後推 `intervalDays`
- 使用者可以按「延後一天」
- 首頁提醒依照到期時間排序
- 過期提醒不要恐嚇，只顯示「稍微延後了」這類語氣

### 2. 紀錄與提醒連動

當使用者新增一筆「換水」紀錄時：

- 若該魚缸有啟用換水提醒，應自動更新該提醒的 `lastDoneAt` 與 `nextDueAt`

當使用者新增一筆「餵食」紀錄時：

- 若該魚缸有啟用餵食提醒，也可以更新下一次提醒

第一版可先支援換水、餵食，之後再支援拍照、清潔、觀察。

### 3. 更寵物化的魚隻頁

目前魚隻只顯示在魚缸卡片內。下一步可做：

- 點擊魚隻卡片進入個體頁
- 顯示名字、魚種、個性、入缸日期、喜歡食物、常待位置
- 顯示與該魚相關的紀錄時間軸

### 4. 社群發文感

目前同步到社群只是簡單貼文。未來可以加入：

- 貼文模板：今日缸照、今日餵食、魚魚近況、狀態觀察
- 公開 / 私人切換
- 可從紀錄自動生成貼文文案
- 標籤選擇

## 文案風格

請使用繁體中文。

語氣應該是：

- 溫和
- 日常
- 有水族專業感，但不要教訓人
- 不恐嚇新手
- 不把魚只當數據
- 鼓勵長期觀察

可接受文案例子：

- 「先留下今天的狀態，之後回頭看才知道魚缸怎麼變穩。」
- 「這缸的換水節奏稍微延後了，今天可以順手看一下魚況。」
- 「備註不是考試卷，今天只想按一下也可以。」
- 「不是只有活著，而是被好好看見。」

不建議文案：

- 「請立刻下藥」
- 「此症狀代表 XX 疾病」
- 「未換水將造成死亡」
- 「最佳繁殖產量」

## 訂閱方向，暫不實作付款

未來可做 Pro 訂閱，但目前原型不要接金流。

Pro 可能包含：

- 不限魚缸
- 不限魚隻檔案
- 完整照片時間軸
- 自訂提醒
- 私人日記
- 雲端同步
- 貼文模板
- 月報

目前只需要在 UI 裡保留概念，不要做真正付款。

## 測試重點

完成任何修改後，請確認：

1. 可以在 GitHub Pages 路徑下正常載入，不依賴根網域 `/`
2. iPad / iPhone Safari 可正常操作
3. 新增紀錄備註可留空
4. 自訂提醒可建立、完成、延後、關閉
5. 新增換水紀錄會更新該缸的換水提醒
6. localStorage 舊資料不存在時能正常使用 seed data
7. localStorage 舊資料格式不完整時不要讓畫面壞掉
8. PWA 快取更新後不會一直卡舊版

## 開發限制

目前請保持靜態前端，不要引入複雜 build tool。

除非另有要求，先不要改成 React / Vue / Flutter。這一版目標是快速驗證流程與產品感。
