# 給 Codex 的任務指令

請接手這個靜態 PWA 原型，專案名稱是「河憩缸誌」。

這是養魚日常紀錄 App，不是水族疾病診斷 App，也不是繁殖管理 App。請保持產品方向：把魚當寵物照顧，著重日常紀錄、魚隻個體、提醒與輕社群互動。

## 本次請優先完成

請在現有 HTML / CSS / Vanilla JS 架構中，加入「自訂提醒系統」。不要導入 React、Vue 或 build tool。

### 功能需求

1. 新增一個提醒設定區，可以放在「我的」頁或新增「提醒」區塊。
2. 使用者可以針對單一魚缸建立提醒。
3. 提醒類型包含：換水、餵食、拍照、清潔、觀察。
4. 使用者可以設定：
   - 魚缸
   - 提醒類型
   - 每幾天一次
   - 提醒時間
   - 備註，可留空
   - 是否啟用
5. 首頁顯示提醒清單，依到期日排序。
6. 每個提醒可以按：
   - 完成
   - 延後一天
   - 關閉
7. 按完成後，自動更新下一次提醒日期。
8. 新增「換水」紀錄時，如果該魚缸有啟用換水提醒，要自動更新該提醒。
9. 新增「餵食」紀錄時，如果該魚缸有啟用餵食提醒，也自動更新該提醒。
10. 保持備註非必填。

### 資料結構建議

```js
{
  id: crypto.randomUUID(),
  tankId: '...',
  tankName: '90 溪流缸',
  type: '換水',
  intervalDays: 7,
  time: '21:00',
  lastDoneAt: '2026-06-08',
  nextDueAt: '2026-06-15',
  enabled: true,
  note: ''
}
```

請注意舊版 seedData 裡的 reminders 只有 title / due，請提供 migration，避免舊 localStorage 直接壞掉。

### UI 風格要求

- 使用繁體中文。
- 語氣要溫和，不要恐嚇。
- 不要寫「不替代診斷」、「不推用藥」這類標籤在畫面上。
- 不要提供藥物建議。
- 不要做疾病診斷。
- 不要把繁殖變成主要功能。
- 備註欄一律可以留空。

### 驗收條件

完成後請確認：

- 開啟 `index.html` 可正常測試。
- 放在 GitHub Pages 子路徑也能正常載入。
- 首頁提醒會顯示今天、明天、已延後等狀態。
- 建立換水提醒後，新增換水紀錄會把下一次換水日期往後推。
- 建立餵食提醒後，新增餵食紀錄會把下一次餵食日期往後推。
- 按「完成」會更新下一次提醒。
- 按「延後一天」會把 nextDueAt 加一天。
- 按「關閉」不再顯示於首頁主要提醒。
- localStorage 沒資料、舊資料、資料不完整時都不會白畫面。
